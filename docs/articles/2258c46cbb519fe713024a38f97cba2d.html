NTFS3 驱动在 Linux 7.2 中获得错误修复与小幅改进

发布于 2026 年 6 月 24 日下午 2:11（美国东部时间）

新的 NTFS 文件系统驱动已合并到 Linux 7.1 中，并在 Linux 7.2 中获得了更多改进。至少目前，NTFS3 内核驱动仍在持续维护，并带来了新的修复和改进。该驱动是几年前在疫情期间被上游合并到 Linux 内核中的。

由于新的 NTFS 驱动仍在证明自身价值，目前没有立即将 NTFS3 从 Linux 内核中移除的计划。Paragon Software 的 Konstantin Komarov 继续维护这个 Linux 内核的 NTFS 读写驱动。对于 Linux 7.2，主要是小幅错误修复，从越界读取到编译器警告修复，以及其他大多属于常规性质的改动。

新增内容包括防止可能栈溢出的保护机制、额外的验证、清除超出有效数据长度的陈旧页缓存，以及其他底层改动。
