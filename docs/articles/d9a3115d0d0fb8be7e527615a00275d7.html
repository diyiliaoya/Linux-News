Canonical 新 AI 工具：让你与 Ubuntu 对话而非打字

Canonical 推出名为 Myna 的 AI 语音转文字工具，完全在本地硬件上运行，无需联网。该工具将随 Ubuntu 26.10 首发，基于 GNOME on Wayland 构建，采用按键通话机制，支持三种大小的语音模型，可适配 NVIDIA GPU、Intel NPU 或 CPU。音频数据仅暂存于内存缓冲区，会话结束后即丢弃。目前项目仍在早期开发阶段，Canonical 正征求用户反馈。

Canonical 新 AI 工具：让你与 Ubuntu 对话而非打字

Myna 完全在本地硬件上运行，将随 Ubuntu 26.10 首发。

今年四月，Canonical 的 Jon Seager 阐述了 Ubuntu 中处理 AI 的框架。该框架将功能分为两类：一类是静默改进现有使用的隐式功能，另一类是你会有意主动调用的显式功能。当时，Jon 将语音转文字和文字转语音作为隐式功能的示例之一。几周后，这一拼图中的一块以 Myna 的形式出现。

虽然该工具仍处于开发早期，但计划随 Ubuntu 26.10（预计十月发布）首次亮相。

AI 驱动的无障碍功能

Canonical Ubuntu 桌面工程总监 Jean-Baptiste Lallement 发布了公告，称语音听写已成为现代平台的常见功能。

对于 Ubuntu 26.10，Myna 的初始版本预计将是一个基于 GNOME on Wayland 的桌面听写工具，采用按键通话机制来控制麦克风何时接收输入。使用时，按住热键、说话、然后松开。一个小活动指示器会在监听时显示，转录的文本会出现在听写开始时光标所在的位置。

识别过程发生在一个名为 Canonical Inference Snap 的沙盒化组件内部，该组件管理会话，而一个音频处理模块处理麦克风拾取的内容，在将其送入模型之前进行降噪和分块。

该 Snap 旨在携带三种大小的语音模型，以及匹配运行 Myna 硬件的运行时环境——无论是 NVIDIA GPU、Intel NPU，还是仅 CPU。

在你大喊“我的数据会被发送到云端服务器！”之前，请注意：语音识别将在本地进行，一旦安装了相应的模型，就不需要互联网连接。

此外，文本只有在最终确定后才会出现，因此你不会看到像某些助手显示实时字幕那样闪烁的半成品单词。音频数据也不会保留，它存储在一个小型内存缓冲区中，会话结束后立即丢弃。

像密码字段听写、唤醒词、持续监听、语音助手、语音命令、翻译、说话人识别和自动语言检测等功能目前均不支持。

这些功能都尚未锁定。该仓库目前仅包含一个许可证、一个 README 文件以及一个存放文档和架构规范的文件夹。

根据过去功能在 Ubuntu 临时版本中的发布方式，我们可能会在 Ubuntu 26.10 的每日构建中看到 Myna。

你还应该知道，Canonical 正在 Myna 规范最终确定前征求反馈，特别是来自已在 Linux 上依赖听写或辅助工具的用户。
