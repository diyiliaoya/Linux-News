Linux WMI 驱动准备支持基于 ACPI 的 ARM64 笔记本电脑

发布于 2026 年 7 月 17 日 上午 09:45 美国东部时间。

Linux 开发者 Armin Wolf 今天发送了一组补丁，用于启用 ACPI Windows 管理规范（WMI）驱动对 AArch64 的支持，使其不再局限于 x86/x86_64 架构。这是朝着在 Linux 上通过 ACPI 支持现代 Windows on ARM 笔记本电脑这一长期目标迈出的一步。

回顾大约 2019 年的 aarch64-laptops 项目，当时已知一些 Windows 10 ARM 笔记本电脑像 x86_64 笔记本电脑一样使用了 WMI。多年来，一直有尝试让 Linux WMI 驱动能够在 AArch64（ARM64）上构建，而不仅限于 x86/x86_64 架构，但这些尝试尚未被主线接纳。

Armin Wolf 创建了一组新的补丁，旨在将 WMI 驱动引入 ARM64，以改善 Linux 对现代 ARM 笔记本电脑的支持。他在今天的补丁中解释道：

“aarch64-laptops 项目发现，一些 Windows on ARM 笔记本包含 WMI-ACPI 映射设备（PNP0C14），其工作方式与 x86 上完全相同。然而，由于 WMI 驱动核心仅支持 x86，这些设备目前不受支持。

第一个补丁移除了这一限制，允许 WMI 驱动核心在所有支持 ACPI 的架构上运行。第二个补丁更新了 acpi-video 驱动，以利用对 x86 以外架构的改进支持。最后一个补丁使通用 WMI BMOF 驱动也能支持 aarch64。

所有补丁已在基于 ACPI 的 AArch64 虚拟机上测试，运行良好。”

希望这次对 AArch64 的 WMI 驱动支持能够进入上游内核。

如果你错过了，几周前还有一些其他相关新闻，其中提出了一种 DeviceTree-ACPI 混合模式，用于改善 Linux 上高通骁龙笔记本电脑的支持。文中还提到，未来的 Windows on ARM 骁龙笔记本电脑将理想地仅使用 ACPI，而不需要设备树。
