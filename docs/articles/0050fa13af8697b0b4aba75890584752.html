Slack Who? 这款自托管聊天应用刚刚开源

由一位开发者用过去一年时间打造，Chatto 是一款可以自托管的团队聊天解决方案。

当大多数团队在寻找聊天应用时，首先想到的解决方案通常是 Slack。它以功能丰富、快速和持续更新而闻名，成为全球数百万团队的首选。

但无论它有多好，它并不是开源的。🙃

幸运的是，开源领域并不缺少替代品。

Rocket.Chat、Mattermost 和 Zulip 都在希望拥有数据自主权的团队中建立了忠实的追随者。

现在，这个列表里又添了一个新名字：Chatto，而且它仍处于开发周期的早期阶段，稳定的 1.0 版本尚未发布。

在过去的几个月里，Hendrik 一直在开发 Chatto，现在他决定以 Apache-2.0 许可证（某些组件有例外）的形式将该项目完全开源。

Chatto 被构建为一种自托管选项，面向那些原本会使用 Slack、Teams 或 Discord 的团队。Hendrik 希望它资源占用轻量、足够简单，管理员只需运行可执行文件即可开始使用。

它涵盖了聊天应用应有的基本功能，包括频道、房间、文件共享、视频嵌入，以及用于精细控制用户访问权限的角色和权限系统。

你还可以获得屏幕共享功能，以及每次语音和视频通话的端到端加密。可加入通话的人数取决于你服务器的容量。

目前没有专门的桌面或移动客户端。路线图将这两项列为可能开发的内容，但它们仍处于探索阶段，这意味着团队还在斟酌是否提供这些客户端。

除了客户端应用，路线图还列出了专门的 Slack 到 Chatto 迁移工具，以及符合 GDPR 标准的数据导出、线程锁定和线程删除等计划中的功能。

Hendrik 目前正在开发新的房间类型、带有专用 API 令牌的机器人账户、Chatto 中心集成以及服务器范围的用户封禁。接下来计划的功能包括管理员紧急锁定模式、慢速模式、应用内消息举报和服务器邀请。

对于那些希望跳过自托管的用户，Chatto 的托管服务也即将进入公开测试阶段。它将运行在欧洲拥有的基础设施上，并提供自动扩展和夜间备份等便利功能。

也不必担心供应商锁定问题，因为数据导入和导出都将变得便捷。
