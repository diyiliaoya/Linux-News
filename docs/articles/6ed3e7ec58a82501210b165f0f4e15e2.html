Ubuntu 26.04 修复清空回收站对话框默认取消的 Bug

Ubuntu 26.04 LTS 修复了一个回收站确认对话框的 Bug，该 Bug 错误地将默认焦点按钮从“删除”改为“取消”，导致用户按回车键时无法清空回收站。该问题由 GNOME 开发者确认是意外更改，修复已合并到上游并打包进 Ubuntu 26.04 LTS，预计在 8 月的 26.04.1 LTS 点发布版中正式生效。

Ubuntu 26.04 LTS 正在修复一个回收站确认对话框的 Bug，该修复将“删除”恢复为默认聚焦按钮，撤销了之前意外将“取消”设为聚焦按钮的更改。

这一“非故意的”焦点切换意味着，你无法再通过按回车键来执行文件删除操作，因为按回车键反而会取消删除。自从升级到 Ubuntu 26.04 LTS 以来，我已经记不清有多少次以为清空了回收站，结果却发现并没有。

我的肌肉记忆一直难以适应需要明确点击（或切换到）删除按钮的操作。

令人恼火，但幸运的是并非有意为之。GNOME 开发者确认按钮焦点是被意外切换的。

现在，一个修复已经合并到上游（https://gitlab.gnome.org/GNOME/nautilus/-/merge_requests/1872），再次将“删除”设为清空回收站对话框中的默认高亮操作。

Ubuntu 已将该修复打包到 26.04 LTS 中，目前位于 proposed 分支进行测试。它应该在 8 月的 26.04.1 LTS 点发布版之前到位，届时从 Ubuntu 24.04 LTS 的升级将正式开放。

从整体来看，这是一个微小的 Bug 修复，鼠标用户可能不会注意到（点击不需要任何焦点）。但对于像我这样习惯按回车键的用户来说，这一改变颠覆了多年养成的操作习惯。

一致性是关键——双关语——有了这个修复，我再也不会意外地让回收站留着一堆垃圾了。
