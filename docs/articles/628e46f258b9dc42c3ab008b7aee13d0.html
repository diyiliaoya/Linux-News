Mozilla 宣布从九月起调整 Firefox 的发布周期，稳定版更新将从每月一次改为每两周一次。此举旨在让发布计划更可预测，并让已就绪的功能更快推送。Mozilla 将此视为一项实验，将监测影响并适时调整。Firefox 154 将是当前周期下的最后一个版本，随后 Firefox 155 于9月1日发布，之后每两周更新一次。尽管节奏加快，未就绪的功能仍会推迟。更快的更新可能帮助 Firefox 与已采用双周更新的 Chromium 系浏览器竞争。现有用户无需任何操作，更新将通过现有渠道自动推送。对于不喜欢频繁更新的用户，Firefox ESR 版本将继续保持较慢的发布节奏，同时定期接收安全更新。Mozilla 表示不会永久维持双周更新，将根据实验结果灵活调整。

Mozilla 将从九月起更改 Firefox 的发布计划，稳定版更新将从每月一次改为每两周一次。

Mozilla 表示，更快的发布节奏将使计划更可预测，并允许已就绪的功能和改进更早推送。

但在给开发者的消息中，

Mozilla 将这一变化描述为一项实验，并表示计划确保其影响是积极的，并在必要时调整发布速度。

Firefox 154 计划于8月18日发布，这将是当前计划下的最后一个稳定版本。随后，Firefox 155 计划于9月1日发布，之后新的稳定版将每两周发布一次。

尽管发布时间表加快，Mozilla 自己的工程师将需要以更快的速度完成工作。尚未就绪的功能将继续开发，直到准备就绪。

选择更快的发布计划可能有助于 Firefox 在与众多基于 Chromium 的竞争对手（如 Google Chrome）的竞争中保持可见的竞争力，这些浏览器已经采用双周更新节奏。

推出更多版本会带来更多 Firefox 用户吗？

该浏览器在主要版本之间已经获得维护更新，因此任何增加都不太可能改变现有用户对更新的看法（因为已经频繁出现“重启 Firefox 以更新”的提示）。这种好处是否会转化为新用户选择 Firefox 而非竞争对手……

不过用户无需做任何事

无论你如何获取 Firefox 更新，你都将获得更多更新。

对于不太喜欢频繁更新的用户，Mozilla 表示，Firefox 扩展支持版本（ESR）渠道将继续以较慢的节奏发布，主要版本之间间隔更长，这些版本会添加重大新功能或涉及大幅设计变更。

ESR 会定期接收安全更新，因此在保持稳定和保持安全之间无需取舍。
