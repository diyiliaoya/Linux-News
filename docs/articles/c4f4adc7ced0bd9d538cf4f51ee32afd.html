这个 Flatpak 运行了 30 年前的 GIMP 版本——早于 GTK

一位爱好者将 GIMP 0.54（1996 年的预览版，早于 GTK 工具包）打包成 Flatpak，可在现代 64 位 Linux 桌面（支持 Wayland）上运行。这个非官方项目托管在 GNOME GitLab 上，目前仍在完善中。GIMP 0.54 是 Linux 吉祥物 Tux 的诞生地，当时使用闭源的 Motif 界面，后来促成了 GTK 的诞生。虽然功能简陋（无图层支持、有 bug），但适合怀旧或复古艺术创作。

GIMP 0.54 Flatpak：在现代 Linux 上运行 1996 年早于 GTK 的构建版本

你是否曾好奇过 1996 年、在 GTK 出现之前的 GIMP 是什么样子？现在，你可以亲自体验了。

有人将 GIMP 0.54 打包成了 Flatpak，可在现代支持 Wayland 的 64 位 Linux 桌面上运行。这似乎是源代码仍可构建的最早版本的 GIMP。

这不是 GIMP 官方的努力，而是一个托管在 GNOME GitLab 上的爱好者项目。它也是一个尚在完善中的古老测试版软件包，维护者承诺未来会添加更多插件和教程。

在进入安装环节之前，先来了解一些充满趣闻的历史，这将有助于解释为什么这个版本比后来功能更丰富的构建版本更值得关注。

一个重要项目的起源

Tux 吉祥物是用 GIMP 0.54 制作的

GIMP 0.54 是 1996 年的早期预览版（1.0 版本直到 1998 年才发布）。当时该应用的界面使用的是 Motif，一个闭源的 GUI 工具包，用其自己的话来说，这使得“向大量用户高效分发变得不可能”。

这种摩擦导致 GIMP 的创始人之一、加州大学伯克利分校的本科生 Peter Mattis 用自己定制的工具包取代了 Motif：GTK，即 GIMP 工具包（在 2026 年搜索这个短语时要小心）。

GTK 从未打算成为一个通用工具包（即大型或专业工具包，更像是一个著名的“自用需求”项目）。然而，在 1997 年，GTK 成为了 GNOME 桌面的基础。

这个版本还有一个特别之处：设计师 Larry Ewing 正是用它绘制了最初的 Linux Tux 吉祥物（完全使用鼠标完成）。没错，就是那个带有抗锯齿边缘的原始 Tux 企鹅。

如何在 Ubuntu 上尝试 GIMP 0.54

一个 30 年前的 GIMP 版本在 2026 年还值得使用吗？作为怀旧之旅或复古艺术创作，当然值得。但你不会用它来做任何硬核或创新性的工作。

GIMP 最初发布时没有其熟悉的吉祥物 Wilber

1996 年，GIMP 0.54 是一个测试版。它存在 bug，没有图层支持，而且那些效果需要通过右键点击画布才能找到。它看起来不像我们大多数人几十年来所熟知的 GIMP，行为方式也完全不同。

令人惊讶的是，它当时确实提供了插件系统——但 Motif 界面的封闭和不灵活性让许多潜在的早期贡献者望而却步。

直到 1998 年第一个 GIMP 1.0 版本发布，才出现了一个能够与当时光栅图像编辑器相媲美的工具阵容。

打包者表示，他们制作这个版本是“为了那些可能对此感兴趣的两个家伙”。

如果我是其中之一……你是另一个吗？

从 GitLab 页面（下方有链接）下载 gimp-0.54.1-10.flatpak。要在 Ubuntu 上安装它，你需要先安装 Flatpak，然后使用终端安装 Flatpak 本身：

flatpak install --user gimp-0.54.1-10.flatpak

该网站表示，如果文本工具无法工作，请尝试安装旧版 X11 字体包。在 Ubuntu 上，这些字体包分布在各个子包中，但运行以下命令即可：

sudo apt install xfonts-base
