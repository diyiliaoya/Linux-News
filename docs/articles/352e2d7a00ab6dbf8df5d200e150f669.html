Git 2.55 默认启用 Rust 支持，新增 git history fixup

发布于 2026 年 6 月 29 日美国东部时间下午 01:50。

Git 2.55 今日发布，作为这款分布式版本控制系统的最新稳定更新。

在此之前，Git 世界中的 Rust 代码在构建时是可选加入的。而在 Git 2.55 中，Rust 代码虽然仍是可选的，但现已默认启用。从 Git 2.55 开始，除非使用 "NO_RUST" 选项明确禁用，否则构建时将默认启用 Rust 支持。不过，Git 3.0 或之后的某个版本可能会强制要求 Rust 支持。

Git 2.55 还进一步改进了增量多包索引的 repack 操作，新增了 git history fixup 子命令，可更轻松地修复之前暂存的提交，以及多项其他底层改进。git history fixup 允许用户直接将暂存的更改合并到较早的提交中，无需经过交互式变基。不过，该子命令在 Git 2.55 中仍被视为实验性功能。
