Khronos 列出首个符合 OpenCL 3.1 标准的实现：运行于 Asahi Linux 的 Apple M1/M2 搭配 Rusticl

Khronos 宣布 OpenCL 3.1 发布仅两个多月后，首个通过一致性测试的实现已出现：运行于 Asahi Linux 的 Apple Silicon M1/M2 图形，搭配 Mesa Rusticl 驱动。这一结果颇具戏剧性，因为 Apple 本身不支持 Linux，而是依靠逆向工程团队和开源贡献者。Rusticl 驱动负责人 Karol Herbst 确认了这一消息，并透露 AMD/radeonsi 和 zink 的提交也在推进中。

OpenCL 3.1 于 2026 年 7 月 14 日美国东部时间晚上 9:14 发布，重点关注 AI 和 HPC 工作负载。仅仅两个多月后，这一对 OpenCL 3.0 的增量更新就有了首个列出的符合 OpenCL 3.1 标准的实现，该实现通过了 OpenCL 3.1 一致性测试套件。它就是运行于 Asahi Linux 的 Apple Silicon M1/M2 图形，搭配 Mesa Rusticl 驱动。

一个颇为奇特的转折是，首个列出的 OpenCL 3.1 符合标准实现是运行于 Asahi Linux 的 Apple Silicon，搭配 Mesa 的 Rusticl 驱动。当然，Apple 本身并不支持 Linux，而是依靠 Asahi Linux 的逆向工程团队和其他开源贡献者。Rusticl 是一个面向 Mesa Gallium3D 驱动的通用 OpenCL 实现。

领导 Mesa Rusticl 驱动开发的 Red Hat 工程师 Karol Herbst 评论道：

“哦，是的……截至昨天，rusticl 是首个 OpenCL 3.1 符合标准的实现。

它在运行 Linux 的 Apple 硬件上正式符合标准，也就是 Asahi Linux。

我可能想完成其他 3.1 提交，AMD/radeonsi 和 zink 已经在推进中，但仍需修复一些烦人的问题。”

因此，一个颇为奇怪的转折是，运行于 Asahi Linux 的 Apple Silicon M1/M2 图形，搭配最新的 Mesa Rusticl 代码，恰好成为首个列出的实现。

对于长期关注 Linux 桌面领域的人来说，可能还记得十多年前，开源 Linux 图形栈在支持较新 OpenGL 和 OpenCL 版本方面一直落后于主要硬件厂商。
