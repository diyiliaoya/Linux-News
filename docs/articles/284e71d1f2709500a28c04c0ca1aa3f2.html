GNU Hurd 在 AArch64 上取得进展，使用 Rust 编写翻译器

发布于 2026 年 7 月 18 日晚上 8:53（美国东部时间）。

GNU Hurd 项目最近发布了 2026 年第二季度状态报告，概述了近期开发工作。Hurd 仍在持续发展！

GNU Hurd 2026 年第二季度进展报告于本月早些时候发布，但直到昨天，其项目网站才修复并恢复了 RSS 订阅。虽然不如 Linux 开发速度那样显著，甚至不如 Haiku 和 Redox 等其他项目，但 GNU Hurd 开发者仍在坚持不懈地推进开发。

过去一个季度中，Hurd 的一些工作包括：宣布为 Hurd 开发 9P 翻译器，其动机是用于浏览文件，并可能通过 virtio-9p 在虚拟机与宿主机之间共享目录树。Hurd 还进行了 msync 验证工作，以更好地遵循 POSIX 规范。此外，还允许特权用户设置其任务优先级 nice 值。

推进 AArch64 支持的工作。在第二季度，还有将更多 AArch64 相关工作上游合并到内核的活动。

Hurd 在第二季度还开展了使用 Rust 编程语言实现 TRIVFS 的工作。这项工作尚未完成，但展示了用 Rust 编写 Hurd 翻译器的可能性。更多细节可参见相关报告。

其他一些活动包括 Neovim 的移植、OpenBSD 的 OpenNTPD 的持续移植工作、Glibc 增强以及其他移植活动。
