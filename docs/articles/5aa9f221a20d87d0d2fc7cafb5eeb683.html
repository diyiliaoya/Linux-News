Audacity 4.0 Beta 让你测试其全新（更美观的）Qt 界面

Audacity 4.0 首个公开测试版发布，带来了数十年来最大的界面变革：从 wxWidgets 迁移至 Qt。新版 UI 更现代化，并引入频谱编辑、可定制工作区、多项目支持、云端保存等功能。但底层音频引擎仍基于旧代码库，通过 au3wrap 模块连接新前端。测试版稳定性不足，建议备份项目后再尝试。

Audacity 4 的首个公开测试版于本月发布，带来了这款标志性开源音频编辑器数十年来最大的设计变革。

该音频编辑器的界面自项目启动以来一直基于 wxWidgets 构建，现在则运行在 Qt 上。不过，处理文件 I/O、项目存储和内置效果的音频引擎仍使用旧代码库，通过一个名为 au3wrap 的模块连接到新前端。

从某种意义上说，Audacity 4 拥有全新外观，但它仍基于相同的核心引擎代码库（尽管经过了精简和清理）。GitHub 更新日志将其描述为使用 Qt 的“从头重写”，但这仅涉及 UI——这并非一个全新的 Audacity。

不过，这确实是一次重大更新。新外观（及其新标志）在很大程度上回应了关于 Audacity 看起来过时的批评。

不仅仅是新外观

除了视觉变化，还有许多新功能和改进值得探索。

频谱编辑与更可定制的 UI

重新设计的包络工具通过拖放和重塑控件处理增益曲线；频谱编辑让你直接在频谱图中选择频率区域，以便对特定频段应用效果；新的元数据编辑器可在导出前处理曲目标题、艺术家等标签。

前导录音功能也首次亮相（以前称为“插入与滚动”）。这让你可以在不影响后续内容的情况下录制音频的某个部分。

正在处理多个项目？现在它们可以同时保持打开状态，并且你可以在项目之间复制片段。Nyquist 和 VST3 插件在所有平台上均受支持，macOS 上支持 AudioUnits，Linux 上支持 LV2。

此外，还有通过 Audio.com 实现的内置云端保存（除非你禁用欢迎对话框，否则它会不断提醒你），一些新的布局选项（可自定义的工作区、彩色片段等），以及一个用于管理项目的新仪表板。

Audacity 3.x 兼容性如何？

Audacity 4.0 的新标志

你可以在 Audacity 4 中打开 Audacity 3 的项目文件，片段、轨道和标签将完整保留。没有迹象表明反向操作可行；考虑到变化之大，更安全的做法是假设 Audacity 4 的项目文件无法在 3.x 中打开。

如果你计划在 4.0 中尝试 3.x 项目并进行更改后保存，请记住这一点。请务必备份你的项目。

Audacity 3.x 的许多功能已包含在 4.0（Beta 2）中。

速度轨道、LADSPA 插件和 Vamp 分析器已被移除。目前尚未说明它们是否会在稳定版发布前回归。如果你需要这些功能，请继续使用稳定且受维护的 3.x 分支。

Audacity 3.7.1 与 4.0 Beta 一同发布。这是一个维护版本，修复了多声道 FLAC 文件导入、效果预设加载，以及在 Linux 上提供了在 HiDPI 显示器上显示效果更好的 wxGTK 界面。

我可以测试 Audacity 4.0 Beta 吗？

你可以从官方网站或项目 GitHub 页面下载适用于 Windows、macOS 和 Linux 的 Audacity 4.0.0 Beta 2。对于后者，有一个 Intel/AMD 64 位 AppImage 可供尝试，但没有 ARM64 版本（Windows 版有一个）。

作为测试版，无法保证稳定性。我只是为了截几张图而随便探索了一下，就遇到了几次严重崩溃，因此这个版本不适合处理任何你无法承受损失的内容。
