Graviton5 CPU 基准测试：几何平均性能较 Graviton4 提升 30%

2026 年 7 月 9 日下午 1:53（美国东部时间）

在去年 12 月首次宣布 Graviton5 之后，近日 AWS 终于让 M9g 和 M9gd 实例普遍可用，这些实例是首批采用这些新型自研 ARM 服务器处理器的 EC2 云实例。Graviton5 采用 Arm Neoverse-V3 核心（Graviton4 采用 Neoverse-V2），支持最多 192 个核心，主频高达 3.3GHz，而上一代 Graviton CPU 的主频为 2.8GHz。以下是 Graviton5 处理器相对于 Graviton4 性能的初步观察。

Graviton5 采用 Arm Neoverse-V3 Armv9.2-A 核心，提供 3.3GHz 主频、192 个核心、更大的 L3 缓存、PCIe Gen6 支持，以及相对于基于 Neoverse-V2 的 Graviton4 处理器的其他增强功能，后者已上市两年。与 Graviton4 一样，Graviton5 仍采用 12 通道 DDR5 系统内存，但速度提升至 DDR5-8800，而 Graviton4 为 DDR5-5600。

Amazon Web Services 宣称，搭载 Graviton5 的 M9g 实例相比基于 Graviton4 的 M8g 实例，计算性能提升高达 25%。AWS 还宣称，M9g 在数据库性能上提升高达 30%，Web 应用性能提升高达 35%，机器学习工作负载性能提升高达 35%。出于对 Graviton5 性能的好奇，自从这些新的 EC2 实例正式可用以来，我最近一直在进行一些基准测试。

由于 AWS 未提供任何评测访问权限，测试成本较高，本轮测试仅对比 Graviton4 M8g 与 Graviton5 M9g。希望了解历代 Graviton CPU 基准测试数据的读者，可以查看 Phoronix 上之前的 Graviton1 到 Graviton4 基准测试。对于那些好奇 Graviton5 与云端 AMD EPYC 和 Intel Xeon 相比如何的读者，如果读者兴趣足够支撑云成本，这些基准测试将在未来几周内以 Phoronix 的单独文章形式发布。

M8g Graviton4 和 M9g Graviton5 的基准测试在 Ubuntu 26.04 LTS 上进行，使用 Linux 7.0 内核、GCC 15 以及该当前 Ubuntu LTS 版本的其他默认配置。鉴于 16 vCPU 大小似乎是不同云提供商中最常见的实例/虚拟机大小，本次 M8g 和 M9g 测试采用“4xlarge”规格，为 Graviton4 和 Graviton5 均提供 16 个 vCPU/核心。4xlarge 规格将 16 个 vCPU 与 64GB 内存配对。

在俄亥俄区域，m8g.4xlarge 按需定价为每小时 0.78272 美元。同一区域 m9g.4xlarge 按需定价为每小时 0.71808 美元。在 4xlarge 规格下，选择 Graviton5 而非 Graviton4 需支付约 9% 的溢价。
