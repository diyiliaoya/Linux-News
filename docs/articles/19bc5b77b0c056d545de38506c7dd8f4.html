Linux补丁终于允许通过蓝牙监控Apple Magic Keyboard/Magic Mouse电池

Linux内核此前仅支持通过USB连接时报告Apple Magic Keyboard和Magic Mouse的电池电量，蓝牙连接时电量显示为0%。开源开发者Alec Hall发布了一组补丁，通过将电池读取操作移至工作队列，解决了蓝牙连接下的电池报告问题，并支持Magic Trackpad 2。补丁目前正在审查中，有望进入主线内核。

Linux补丁终于允许通过蓝牙监控Apple Magic Keyboard/Magic Mouse电池

发布于2026年7月15日，美国东部时间上午8:54。

除了在Linux上启用新Apple Silicon SoC支持的持续挑战外，Linux上的Apple外设支持因产品而异，情况好坏参半。现在正在解决的最新功能是，当Apple Magic Mouse和Magic Keyboard通过蓝牙连接时，能够实现电池报告功能。

Linux已经支持输入设备的电池报告，但限制之一是电池报告仅在通过USB连接时有效……这并不完全有用，因为通过蓝牙无线连接时，电池电量才更为重要。当前的Linux内核版本在通过蓝牙连接Apple设备时会报告0%的电量，这显然毫无帮助。

开源开发者Alec Hall发布了一组补丁，使Apple Magic Keyboard、Magic Mouse 2和Magic Trackpad 2能够通过蓝牙进行电池报告。Alec在解决这一显著缺陷时解释道：

“Apple的Magic Keyboard和Magic Trackpad 2 / Magic Mouse 2会向主机报告电池电量，但内核仅通过USB显示该信息；通过蓝牙连接时，power_supply始终显示为0%，尽管这些设备几乎始终通过蓝牙使用。

两个驱动程序都已经获取电池信息（针对输入报告0x90的GET_REPORT），但仅通过USB进行。在蓝牙上直接启用会导致机器硬锁定：该获取操作在定时器回调（原子上下文）中运行，而uhid传输在hid_hw_request()内休眠。因此，每个驱动程序的补丁在启用蓝牙之前，将获取操作移至工作队列。最后一个补丁教会hid-magicmouse从设备的状态字节中报告充电状态。

补丁3解释了一个供应商状态字节，其位含义是通过经验确定的；如果审查者偏好，我可以将其重写为报告描述符修复，以暴露HID_BAT_CHARGING。

已在Magic Keyboard 2021和Magic Trackpad 2上通过BlueZ（uhid）进行了测试。”

这些补丁目前正在审查中，有望使这些hid-magicmouse和hid-apple驱动程序的改进进入主线Linux内核。
