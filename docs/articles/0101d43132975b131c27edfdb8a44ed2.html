Linux 7.2 中，F2FS 文件系统集成了 FSERROR 错误报告框架，实现了 I/O 错误报告标准化。同时，通过移除运行验证用的内存数据结构（如内存块位图和 SIT 版本位图），显著降低了内存占用。此外，还支持每 I/O 写入流、FDP 流映射等优化。

F2FS 集成 FSERROR 报告功能，在 Linux 7.2 中减少内存占用

发布于 2026 年 6 月 24 日 美国东部时间下午 12:48。

闪存友好文件系统（F2FS）的变更已合入 Linux 7.2。

Linux 7.2 中 F2FS 最显著的变更之一是 FSERROR 错误报告支持。FSERROR 框架在 Linux 7.0 中被合并，用于处理通用 I/O 错误报告，该功能在 Linux 文件系统中可以通用，而无需每个文件系统自行实现报告机制。现在，随着 Linux 7.2 的发布，F2FS 已加入 FSERROR 阵营，实现了 I/O 错误报告的标准化。

Linux 7.2 中的 F2FS 还通过还原用于运行验证的内存数据结构，减少了使用该文件系统的内存占用。此外，还有各种其他改进。F2FS 现在支持对直接写入操作遵循每 I/O 写入流，将数据写入映射到 FDP 流，以及其他优化。

内存块位图在存在十年后被移除，但在此期间开发者从未发现任何内存损坏的报告案例，因此它只是在浪费内存。内存中的 SIT 版本位图在 F2FS 中也存在了大约十年，同样未证明有任何实际价值，因此也被移除以节省内存。

关于 F2FS 功能变更以及针对这一闪存优化文件系统的各种错误修复的更多详细信息，可通过已合入 Linux 7.2 的补丁获取。
