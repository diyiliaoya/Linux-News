Graviton5 性能超越 Intel Xeon Granite Rapids，但不及 AMD EPYC Turin

AWS 新推出的 Graviton5 处理器在性能上超越了 Intel Xeon Granite Rapids，但相比 AMD EPYC Turin 仍有差距。本文通过对比 m9g.4xlarge（Graviton5）、m8g.4xlarge（Graviton4）、m8a.4xlarge（AMD EPYC Turin）和 m8i.4xlarge（Intel Xeon Granite Rapids）四种实例的基准测试数据，分析了 Graviton5 在 ARM 服务器领域的竞争力。测试在 Ubuntu 26.04 LTS、Linux 7.0 内核和 GCC 15.2 环境下进行，结果显示 Graviton5 在性价比上具有一定优势，但绝对性能仍落后于 AMD EPYC Turin。

Graviton5 性能超越 Intel Xeon Granite Rapids，但不及 AMD EPYC Turin

发布于 2026 年 7 月 13 日 12:53 PM EDT。

在 AWS M9g 系列（首批搭载全新 Graviton5 CPU 的实例）近期正式发布后，我最近进行了基准测试，对比了 Graviton4 与 Graviton5 的 CPU 性能。新一代 AWS Graviton 处理器在从 Arm Neoverse-V2 核心升级到 Neoverse-V3 核心、内存从 DDR5-5600 升级到 DDR5-8800 以及其他改进方面，带来了非常可观的代际提升。对于那些想知道 Graviton5 ARM 服务器处理器与 AMD EPYC 和 Intel Xeon 相比如何的人，这里提供来自 Amazon EC2 云的一些额外对比数据点。

作为上周 m8g.4xlarge 与 m9g.4xlarge 基准测试（在相同 vCPU/核心数下对比 Graviton4 与 Graviton5）的补充，现在这里的数据包括了 m8a.4xlarge 和 m8i.4xlarge，它们是 M 系列 EC2 实例类型中最新搭载 AMD EPYC 和 Intel Xeon 的产品。

Amazon EC2 M8a 实例搭载 AMD 第五代 EPYC "Turin" 处理器，使用半定制 AMD EPYC 9R45 SKU，M8a 实例的最高频率为 4.5GHz。M8a 的每个 vCPU 对应一个物理 CPU 核心（无 SMT）。M8a 实例使用 DDR5-6400 内存。测试时，m8a.4xlarge 的按需定价为每小时 0.97376 美元，远高于搭载 Graviton5 的新型 m9g.4xlarge 每小时 0.78272 美元的按需定价。

Amazon EC2 M8i 实例搭载 Intel Xeon 6 Granite Rapids 处理器，即半定制 Intel Xeon 6975P-C。M8i 实例的 Xeon 6 CPU 全核心睿频为 3.9GHz。与 M8g/M9g 和 M8a 实例（每个 vCPU 对应一个物理 CPU 核心）不同，M8i 依赖于物理核心和兄弟线程的混合。因此，m8i.4xlarge 由八个物理 CPU 核心加上其 HT 兄弟线程组成。M8i 实例还使用了 DDR5-7200 内存，尽管 Xeon 6900 系列能够支持 MRDIMM-8800 内存。m8i.4xlarge 的按需每小时定价为 0.84672 美元，比 m8a.4xlarge 实例便宜，但仍高于 m9g.4xlarge，并且物理核心数量只有后者的一半。

所有这些 4xlarge 实例均配备 64GB 系统内存。我在运行 Ubuntu 26.04 LTS（搭载 Linux 7.0 内核和 GCC 15.2 编译器）的情况下，对所有四种实例类型进行了基准测试。
