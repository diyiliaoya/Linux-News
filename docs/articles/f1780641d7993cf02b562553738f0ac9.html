AMD 开始为 Linux 7.3 准备图形驱动变更

发布于 2026 年 7 月 5 日 上午 06:32（美国东部时间）

继 Intel 开始为 Linux 7.3 提交图形驱动补丁后，本周 AMD 也开始向 DRM-Next 发送其“新内容”的拉取请求，用于 Linux 7.3 内核周期。

周四，AMD 的 AMDGPU/AMDKFD 维护者 Alex Deucher 发送了首批拉取请求，包含针对 2026 年晚些时候该内核版本的内核图形/计算驱动变更的新特性材料。

Linux 7.3 中 AMDGPU 和 AMDKFD 的一些早期变更包括：实现计算管道重置支持，以便在 GFX11 (RDNA3) 和 GFX12 (RDNA4) 硬件上遇到问题时重置计算管道。还有用户队列 (UserQ) 更新、RAS 更新、从 DisplayID 解析显示面板类型信息的能力、DCN 4.2 IP 更新、GFX 11.7 更新，以及其他各种更新。此外还有针对 8K 显示面板、背光及其他各种问题的修复。

该 C 驱动还包含大量代码加固更新，改进了 vBIOS 解析的边界检查，并增加了其他各种边界检查。AMDKFD 驱动也通过 CRIU 边界检查得到了加固。

此 Linux 7.3 的拉取请求还包括来自 Valve Linux 图形团队 Timur Kristóf 的工作，涉及早期 Radeon GCN GPU 的软重置和 GPU 恢复改进。

AMDKFD 计算驱动现在还允许应用程序在发生致命错误时选择退出 SIGBUS，以及各种修复。ROCm 已经支持 SIGBUS 处理变更。

关于这些针对 Linux 7.3 的早期 AMDGPU/AMDKFD 内核驱动变更的更多细节，可以查看完整的补丁列表。预计未来几周内，还会有更多 AMD 图形驱动变更作为 Linux 7.3 的附加特性材料提交到 DRM-Next。
