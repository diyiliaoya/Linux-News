Linux 7.2-rc1 发布：“情况看起来相当正常”，同时引入 AMDGPU HDMI 2.1 FRL、AMD ISP4 和 CAS

Linux 7.2-rc1 已发布，标志着合并窗口关闭。该版本引入了多项新特性，包括缓存感知调度、USB4STREAM、NTFS 驱动修复、strncpy API 的移除、ARCTIC 风扇控制器驱动、AMD ISP4 驱动以及 AMDGPU HDMI 2.1 FRL 的初步支持。Linus Torvalds 表示情况看起来正常，尽管 AMD 头文件占用了三分之一的补丁。Linux 7.2 稳定版预计在八周后发布，将驱动 Fedora 45 和 Ubuntu 26.10。

正如预期，Linux 7.2-rc1 在不久前发布，标志着合并窗口的结束。现在将进入大约八周的测试期，之后 Linux 7.2 稳定版将发布，进而为 Fedora 45 和 Ubuntu 26.10 等系统提供支持。

Linux 7.2 合并了许多新特性，包括缓存感知调度、各种其他性能优化、英特尔创新的 USB4STREAM 功能、Linux 7.1 中新的 NTFS 驱动修复、经过六年工作后 strncpy API 的移除、新的 ARCTIC 风扇控制器驱动、AMD ISP4 驱动的引入，以及 AMDGPU HDMI 2.1 FRL 支持的初步工作。我详细的 Linux 7.2 特性概述将在未来几天内发布。Linux 7.2 的代码量已超过 4300 万行。

Linus Torvalds 写道：

“两周过去了，合并窗口已经关闭。这个版本的情况看起来相当正常（敲木头），我在下面附上了我的合并短日志，因为我们显然有太多更改无法逐一列出。

统计数据看起来相当正常，尽管另一个 AMD 头文件转储意味着三分之一的补丁只是各种 AMD GPU 寄存器定义。

这本身并不罕见，如果你忽略那部分，其余部分看起来也相当正常：略超过一半的补丁是驱动程序（即使不考虑那个 AMD 寄存器转储，其他 GPU 驱动更改也出现了，但里面各种东西都有一点），其余部分则是常见的架构更新、工具、文档和核心内核更新。”
