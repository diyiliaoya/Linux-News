Linux DT补丁为Apple M3 Pro / Max / Ultra提供非常基础的支持

发布于2026年7月10日 06:26 EDT

上游已合入Linux 7.2内核的是对Apple M3 SoC设备启动Linux的初步支持。但这只是最基础的支持，仅能启动到控制台，尚未提供任何加速图形或其他日常使用M3 Apple设备于Linux上所需的典型功能。现在，这项工作得到了补充：额外的设备树补丁也支持在M3 Pro / Max / Ultra设备上启动Linux。

继Linux 7.2的Apple M3初步启动支持以及一些针对Apple M4 SoC的初始DT补丁之后，现在有一组设备树补丁正在审查中，旨在让Apple M3 Pro / M3 Max / M3 Ultra硬件能够使用主线Linux内核启动。

Asahi Linux贡献者Janne Grunau在解释这个M3 Pro/Max/Ultra启动补丁系列时表示：

“本系列为搭载M3 Pro、Max和Ultra SoC的Apple Silicon设备添加了设备树。M3代设备比其前代M1和M2要少。唯一的非笔记本电脑设备是M3 Ultra Mac Studio。笔记本电脑则是已知的14英寸和16英寸MacBook Pro，现在搭载M3 Pro和M3 Max SoC。核心数较少的M3 Max变体除了内存总线为384位而非完整M3 Max的512位外，还有一个独立的标识符（T6034），因此共有六个笔记本电脑设备树。

与M1和M2 Pro/Max/Ultra的另一个区别是，M3 Pro是一个独立的SoC设计，而非较小的M3 Max。因此，两种M3 Max变体和M3 Ultra将使用“apple,t6030”作为兼容前缀。在M1和M2代中，Pro、Max和Ultra SoC共享“apple,t6000”/“apple,t6020”作为通用前缀。目前没有已知差异，但M3 Pro和M3 Max之间的关系不如以前那么紧密。

本系列添加了与v7.2-rc1中基础M3（T8122）相同级别的硬件支持。这包括CPU核心、中断控制器、电源状态、看门狗、串口、引脚控制器、I2C和启动帧缓冲。这旨在作为基础，以便可以同时为所有基于M3的设备添加对其他硬件的支持。”

对于这些早在2023年11月发布的Apple设备，这是主线Linux支持朝着正确方向迈出的一步。要让Apple M3级别的硬件在Linux上完全可用，具备GPU支持以及所有其他集成设备功能，尤其是对于MacBook笔记本电脑，还需要更长时间。
