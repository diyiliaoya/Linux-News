NetworkManager 1.58 正式发布，新增对 464XLAT 客户侧转换器（CLAT）的支持，允许 IPv6 单栈网络中的设备访问传统 IPv4 服务；同时新增对 GENEVE 覆盖隧道协议的支持，用于虚拟化流量路由。此外，D-Bus API 和 nmcli 工具现在支持跨重启持久化托管状态，WiFi 连接新增 6GHz 频段支持，并改进了 IWD 后端的省电功能。此版本弃用了 Wireless Extensions 和 dhclient 后端。

NetworkManager 1.58 发布，支持 464XLAT CLAT 与 GENEVE 接口

发布于 2026 年 7 月 20 日 下午 4:12（美国东部时间）

NetworkManager 1.58 今日发布，带来了最新的功能增强，以改善 Linux 上的有线和无线网络管理。

NetworkManager 1.58 新增了对 464XLAT 客户侧转换器（CLAT）的支持，用于创建一个虚拟 IPv4 接口，拦截来自传统应用程序的出站 IPv4 流量，并将其头部转换为 IPv6。464XLAT 旨在帮助 IPv6 过渡，使仅支持 IPv6 网络上的设备仍能访问传统的 IPv4 服务和应用程序。

NetworkManager 1.58 还新增了对 GENEVE 接口的支持，该接口用于通用网络虚拟化封装覆盖隧道协议，作为背板在虚拟机管理程序和虚拟交换机之间路由虚拟化流量。

NetworkManager 的 D-Bus API 和 nmcli 命令行工具现在允许跨重启持久化托管状态。

在 WiFi 方面，WiFi 连接的“band”属性现在支持“6GHz”值。当使用 IWD 后端时，powersave 属性现在也能正常工作。

NetworkManager 1.58 弃用了对 Wireless Extensions 的支持，并默认禁用了该功能。它还移除了对 dhclient 作为 DHCP 后端的支持，该功能已在多个版本中被弃用。

关于 NetworkManager 1.58 的下载和更多详情，请访问 FreeDesktop.org GitLab。
