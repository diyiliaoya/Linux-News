下一个 Bcachefs 版本计划在内核模块中包含 Rust 代码

发布于 2026 年 6 月 29 日下午 3:30（美国东部时间）。

Bcachefs 文件系统已经在用户空间工具中使用了 Rust 编程语言，并且他们一直计划在内核模块中也使用 Rust。从下一个 Bcachefs 版本 v1.38.7 开始，他们希望开始在其内核驱动中包含 Rust 代码。

几天前刚刚发布了一些不错的改进，而其主要开发者 Kent Overstreet 已经在为 Bcachefs 1.38.7 规划重大变化。在今天的一篇文章中，他概述了 Bcachefs 1.38.7 的计划，这将是第一个开始在内核 Bcachefs 模块中包含 Rust 的版本。最初这将是一个软依赖，同时他们将处理部署流程并搭建相关基础设施。

在 Bcachefs 内核模块中使用 Rust 的计划确实使 Bcachefs 作为树外模块的处理变得复杂。Kent Overstreet 指出了使用动态内核模块系统（DKMS）构建 Rust 模块所面临的挑战。对于大多数用户来说，DKMS 构建 Rust 模块“不切实际”，因为安装的 Rustc 版本需要与编译内核所用的版本匹配，这是由于 Rust 缺乏稳定的 ABI。

此外，还需要安装相同的 Rust stdlib 源码、bindgen CLI 版本，并且内核构建中的一些 Rust 工件也必须存在。因此，Bcachefs 最终计划面向更常规的发行版二进制包来提供内核模块，而不是继续使用 DKMS 构建。这对于任何其他包含树外 Rust 内核模块的项目来说，都将是一个类似的挑战。
