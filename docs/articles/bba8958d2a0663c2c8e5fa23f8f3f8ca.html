TUXEDO OS 弃用 Ubuntu，转向永不进入稳定版的 Debian 基础

TUXEDO Computers 宣布其 Linux 发行版 TUXEDO OS 将从 Ubuntu LTS 基础迁移至 Debian 的测试分支（Testing），并永久不跟随 Debian 稳定版。此举源于对 Ubuntu 的 Snap 依赖、老旧基础库、AI 路线图不明确及安全更新缓慢的不满。新系统将默认使用 Btrfs 文件系统和自动快照，并计划进行视觉革新。现有用户需全新安装，迁移指南将在最终版发布前提供。

TUXEDO OS 弃用 Ubuntu，转向永不进入稳定版的 Debian 基础

他们对 Ubuntu 的依赖“ snapped ”了，所以现在正在“测试” Debian。

TUXEDO Computers 已宣布其 Linux 发行版将从 Ubuntu 迁移，由 Debian 取代其新基础。TUXEDO OS 是每台 TUXEDO 机器的默认选择，但并不局限于其硬件，任何人都可以获取镜像并在自己的电脑上运行。

未来，该发行版将基于 Debian 的测试分支（Testing），而非 Ubuntu LTS 版本，公司称此为“永不进入稳定版”策略。这一转变也是永久性的，TUXEDO OS 将不会跟随下一个 Debian 稳定版。

这一切换背后，也包含了对 Canonical 近期引导 Ubuntu 方向的不满。在宣布重新选型时，他们表示：

通过迁移到 Debian，TUXEDO OS 获得了显著更高的独立性，同时减少了维护最新软件所需的工作量。其结果是一个稳健的操作系统，明确聚焦于数字主权——无论是对于 TUXEDO 客户，还是在第三方硬件上运行 TUXEDO OS 的用户。

来源：TUXEDO Computers

TUXEDO 表示，随着时间推移，老旧的 Ubuntu 基础使得向后移植（backporting）越来越困难，因为较新的依赖项常常缺失或停留在过时版本。当核心库（如 KDE Plasma 所依赖的 Qt）更新并破坏从 Ubuntu 仓库拉取的软件时，情况会变得更糟。

Snap 是另一个痛点，因为 Canonical 不断推进仅以 Snap 形式打包和交付软件，使得 TUXEDO 更难将这些组件排除在发行版之外。

此外，Ubuntu 的 AI 路线图并未提供关于其实际运作方式的明确信息，而缓慢的安全更新则是最后的争议点。

随着 Ubuntu 的退出，Debian 的开发分支 Testing 成为新的基础。新软件包从 Sid（不稳定版）进入 Testing，前提是它们能证明从相同源码构建出一致的结果——这是 Debian 在五月强制实施的要求。

这一切换在底层也已可见，因为 TUXEDO OS 的内部测试版本正在从 Debian 的仓库而非 Ubuntu 的仓库拉取软件包。

不要认为这仅仅是重新选型；TUXEDO Computers 还在着手引入一些重大升级。

Btrfs 将成为新安装的默认文件系统，并搭配 snapper 用于每次更新前的自动快照。这与 openSUSE 多年来使用的设置相同，因此是一个经过验证的解决方案。

他们尚未详细说明内核策略。我的猜测是——在 TUXEDO 确认之前——会更接近 Ubuntu 或 Fedora 的处理方式：快速推送较新内核，而不是为了稳定性而停留在较旧的内核上。

视觉革新也在进行中，不过他们尚未锁定新外观，当前构建仍在新的 Debian 基础上运行旧主题。游戏和企业用例也都在关注之中，具体细节将在稍后分享。

如果你已经在运行基于 Ubuntu 的 TUXEDO OS 3，这里没有直接的升级路径。需要进行全新安装，TUXEDO 表示在最终版本发布前会提供完整的迁移指南。

另外，如果你更愿意留在 Ubuntu 基础上，该公司将提供一条过渡路径到 Ubuntu LTS。

广泛的 Beta 测试阶段将在未来几周内启动，面向非生产环境。预计会根据反馈和沿途出现的任何发布阻塞问题进行调整。

如果你想更全面地了解他们正在筹备的内容，下一届在八月举行的 TUXEDO 活动将是好去处。TUXEDO Computers 团队将在专门的演讲中展示这一新的开发方向。
