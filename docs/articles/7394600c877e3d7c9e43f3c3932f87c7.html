Ubuntu 26.10 将在设置中添加 Ubuntu 认证检查功能

Ubuntu 26.10 将在“设置 > 系统 > 关于”页面新增“检查此电脑是否通过 Ubuntu 认证”选项，用户可通过图形界面而非命令行查询硬件认证状态。该功能会读取 Canonical 硬件 API 的本地缓存结果，避免每次打开都访问服务器。状态包括已认证、未认证、已为其他版本认证、未认证但存在类似认证型号等。用户还可选择定期检查，仅发送硬件信息，不含个人数据。该功能已上传至每日构建版，预计在 10 月中旬稳定版中正式推出。

如果你拥有官方 Ubuntu 认证设备，这个新面板可以帮助你了解自己的硬件配置是否与某个认证型号相似。

设备要作为官方 Ubuntu 认证产品出售，ODM 或 OEM 需要向 Canonical 支付许可费，以针对其搭载的 Ubuntu 版本验证硬件。

这笔费用涵盖了 Canonical 进行的额外测试和维护工作，有时还包括构建定制 OEM 镜像，并有助于保持更新和硬件支持的持续运行。

我们大多数人都在没有闪亮“认证”标签的设备上运行 Ubuntu，而且它们运行良好。认证状态并不意味着设备比未认证的设备运行得更好，只是表明 OEM 为测试和持续维护付费，以确保其运行良好。

本周已上传，因此未来几周内，Ubuntu 26.10 每日构建版本的用户将能看到该功能——在 10 月中旬稳定版发布之前。
