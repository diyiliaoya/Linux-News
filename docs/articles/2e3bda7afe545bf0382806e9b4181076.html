Ubuntu 风味版现在必须发布 Beta 版本才能正式推出

Ubuntu 宣布了一项重要政策更新，要求所有 Ubuntu 风味版必须发布 Beta 版本，不再允许例外。此前，错过 Beta 截止日期的风味版仍可申请一次性豁免，但 Ubuntu 26.04 LTS 周期中，使用 UKUI 桌面的 Ubuntu Kylin 在错过 Beta 窗口后仍发布了正式版，这促使了政策收紧。Canonical 的 Oliver Reiche 在邮件列表中表示，只有按时提交 Beta 版本的风味版才能获得官方正式发布资格。对用户而言，这意味着更稳定的发布版本，因为 Beta 版本冻结了快照，便于更广泛的测试，且 Beta 与正式版之间的包差异将仅限于 Bug 修复。

Ubuntu 风味版现在必须发布 Beta 版本才能正式推出 - OMG! Ubuntu

Ubuntu 宣布了一项“重要政策更新”，要求所有 Ubuntu 风味版必须发布 Beta 版本，无一例外。

大多数风味版每六个月都会顺利达到 Beta 里程碑。但在此之前，错过截止日期的风味版仍可申请一次性豁免。

在 Ubuntu 26.04 LTS 周期中，这种情况就发生在 Ubuntu Kylin 身上——这是一款面向中国用户、使用 UKUI 桌面的衍生版本。它错过了 Beta 窗口，但仍然发布了最终正式版。

这种情况将不再发生。要获得官方稳定版本，风味版现在必须与其他所有风味版同时发布 Beta 版本，正如 Canonical 的 Oliver Reiche 在邮件列表公告中所说：

“为确保每个风味版都为最终发布做好充分准备，请注意，除非风味版已按预定时间表成功提交 Beta 版本，否则将不被考虑进行官方发布。”

对于风味版的用户来说，这有一个明显的好处：在四月或十月到来时，发布版本将更加稳定。

每日构建版本也允许测试，但往往每天都在变化（并且主要面向同一小部分热心用户）。Beta 版本则冻结一个快照并为其命名，因此更广泛的用户群体可以测试相同的构建、组件和软件包集合。

根据公告，Beta 版本与最终发布版本之间的包差异应最小化，并仅限于 Bug 修复（不过也有官方机制允许后期添加功能，即功能冻结例外）。

但将 Beta 版本作为硬性要求，意味着风味版开发者将确切知道哪些 Bug 需要修复，以及何时修复。
