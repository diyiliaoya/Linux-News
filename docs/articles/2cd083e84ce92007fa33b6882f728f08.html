AUR恶意软件泛滥后，Yay v13让你编写自己的安全网

Arch Linux的AUR（Arch用户软件仓库）近期遭遇多波恶意软件攻击，超过1500个软件包被入侵。流行的AUR助手Yay发布v13版本，新增基于Lua的钩子和配置功能，允许用户编写脚本自动化安全审查。同时，PKGBUILD最后修改时间戳现在直接显示在搜索结果和升级菜单中，帮助用户识别潜在风险包。Yay v13旨在从用户层面应对AUR的安全挑战。

AUR恶意软件泛滥后，Yay v13让你编写自己的安全网

该版本引入了基于Lua的钩子，同时提供更简单的方式查看PKGBUILD最近一次修改的时间。

你可能已经知道，AUR最近经历了一段艰难时期，在Arch开发者能够控制局面之前，超过1500个软件包在三波独立的恶意软件攻击中遭到入侵。

流行的Arch Linux AUR助手旨在从用户层面解决这一混乱局面，引入了新功能，使你在安装前更容易发现风险包，并让你能够自动化审查工作。

检测恶意包的新工具

新的PKGBUILD最后修改时间戳现在显示在方括号内。

搜索结果、yogurt提示和升级菜单现在都带有这些时间戳，显示软件包的PKGBUILD上次修改后过去了多久。这让你提前了解哪些包在安装前可能值得仔细检查。

Yay的维护者澄清说，这个数字本身并不能说明什么。上周编辑过的东西并不自动危险，多年未动的东西也不自动安全。

这只是一个额外的信号，供你在决定安装前权衡。

另一个主要新增功能是支持基于Lua的钩子和配置，让你可以编写脚本控制Yay在安装和升级流程不同阶段的行为。现在你可以在以下位置放置一个文件：

$XDG_CONFIG_HOME/yay/init.lua

~/.config/yay/init.lua

Yay会直接从该文件中读取设置和钩子。

如果完全省略该文件，则不会运行任何与Lua相关的内容。

原有的配置也不会消失，它位于Lua配置之上并可以覆盖已有设置，而你在命令行传递的标志则优先于其他所有配置。

其中一个新钩子在升级过程中间触发，当Yay已经确定需要升级哪些包但尚未将包排除屏幕展示给你时生效。

另外两个钩子在实际安装运行之前生效，只是触发顺序晚于前一个钩子。

另一个钩子在PKGBUILD被获取后立即触发，足够早地在你看到任何菜单之前中止安装。当makepkg --verifysource完成拉取和检查源代码后，另一个钩子触发，此时脚本可以查看PKGBUILD及其下载的实际文件，仍然在安装之前。

除此之外，v13版本还增加了用于过滤搜索结果和在包安装完成后执行操作的钩子。其余部分主要是清理工作，例如恢复缺失的区域设置文件，并且ALPM执行器获得了适当的日志回调和新的Debug方法。

你可以在你的Arch Linux或基于Arch的系统上通过从AUR克隆并使用makepkg构建来运行Yay：

git clone https://aur.archlinux.org/yay.git

cd yay && makepkg -si
