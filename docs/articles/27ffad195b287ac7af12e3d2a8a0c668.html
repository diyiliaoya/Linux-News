AI Agent 可通过 DNS 获得验证身份

Linux 基金会宣布推出开放标准 ANS（Agent Name System），为 AI Agent 提供基于 DNS 的验证身份。每个 Agent 将获得类似 DNS 的名称和版本号，并通过证书和不可篡改的公开日志记录注册、续期和撤销信息。该标准基于现有 IETF 草案，GoDaddy 已提前部署，使用现有基础设施处理超过 1 亿个 SSL/TLS 证书。

AI Agent 可通过 DNS 获得验证身份

该开放标准将每个 Agent 的身份与证书和无人能编辑的公开透明日志绑定。

Linux 基金会表示，它计划推出 ANS（Agent Name System），这是一个开放标准，扩展了 AI Agent 证明自身身份的方式。

在实践中，这意味着能够查找一个 Agent，并检查谁在真正运营它、它被授权做什么，以及其代码或历史记录中是否有任何更改。

引用一篇研究论文，Linux 基金会表示，82% 的高管计划在未来一到三年内采用 AI Agent，而大多数高管在 Agent 投入生产运行后，仍然没有可靠的方法来验证或管理它们。

目前，没有任何机制阻止一个 Agent 声称自己是 support-agent.acme.com，因为没有人能够验证这是否属实。

找到正确的 Agent 同样困难。目前没有任何东西能将 Agent 的名称和能力与一个特定的、经过验证的、你可以实际访问的版本关联起来。ANS 旨在同时解决这两个问题。

在 ANS 下注册的每个 Agent 都会获得一个 DNS 风格的名称，并内置版本号，例如 ans://v1.0.0.my-agent.example.com。只有当 Agent 的域名通过与网站获取普通 TLS 证书相同的 DNS 和 ACME 检查后，该名称才会附带一个身份证书。

每次注册、续期或撤销都会被记录在一个防篡改日志（仅可追加的 Merkle 日志）中，因此在这些事件发生后，任何人都无法编辑 Agent 的历史记录。一个名为 Verifier 的独立离线工具可以在无需实时连接注册表的情况下检查这些记录。

在你认为这是他们新发明的东西之前，事实并非如此。GoDaddy 在 Linux 基金会介入前几个月就已经让 ANS 运行起来，它基于现有的 IETF 草案构建。

GoDaddy 的工程师没有从头构建新的证书和 DNS 系统，而是复用了他们已在生产环境中使用的基础设施。这包括处理超过 1 亿个活跃 SSL 和 TLS 证书的同一证书服务，以及他们现有的 DNS 系统。

GitHub 上的 ANS 组织目前托管着八个仓库。主要仓库是 ans，这是一个采用 MIT 许可证的 Go 代码库，实现了整个技术栈，包括注册表、日志记录和验证器，作为任何人都可以运行的工作参考实现。

当然，现在还为时过早，但据报道，只需安装 Go、openssl、curl 和 jq，整个技术栈大约在 60 秒内就能启动。
