Linux 7.2 特性：缓存感知调度、USB4STREAM、AMD ISP4、AMDGPU HDMI 2.1 FRL

Linux 7.2 内核即将于 2026 年 8 月发布，代码库超过 4300 万行。主要新特性包括：缓存感知调度优化现代 Intel/AMD CPU 的任务放置；USB4STREAM 实现 USB4/Thunderbolt 系统间数据传输；Apple M3 设备可启动主线内核但尚不可用；AMD ISP4 驱动终于进入主线；AMDGPU 初步支持 HDMI 2.1 FRL；以及防止文件系统泛滥的新指南。此外还有 Intel TDX 运行时更新、Zen 6 准备、MGLRU 改进等众多优化。

Linux 7.2 特性：缓存感知调度、USB4STREAM、AMD ISP4、AMDGPU HDMI 2.1 FRL

发布于 2026 年 6 月 30 日 12:20 PM EDT。

正在努力于 8 月发布，其代码库超过 4300 万行。Linux 7.2 带来了许多新变化，正如今天的特性概述所总结的那样。

Linux 7.2 中最大的终端用户亮点包括：合并了缓存感知调度，以帮助在现代 Intel/AMD CPU 上进行任务放置；非常实用的 USB4STREAM，用于使用 USB4/Thunderbolt 的系统间数据传输；Apple M3 现在可以在主线内核上启动，但尚不可用；AMD ISP4 驱动终于进入主线；初步的 AMDGPU HDMI 2.1 FRL 支持；以及防止 Linux 内核中文件系统进一步泛滥的新指南。

继续阅读以获取 Linux 7.2 显著变更的更详尽概述。更多 Linux 7.2 内核性能基准测试正在 Phoronix 进行中。

Apple M3 设备现在可以使用 Linux 7.2 内核启动，但该支持对终端用户而言尚未达到任何实用阶段。

缓存感知调度已合并，用于改善具有多个最后一级缓存（LLC）域的 CPU 上的任务放置。

Intel TDX 现在支持运行时更新，以便在避免重启/停机的情况下应用重要的安全更新，适用于可信域扩展。

- AMD 在音频协处理器方面正在为其下一代音频 IP 准备“重大设计变更”。

为 Zen 6 做更多准备。

AMD ISP4 驱动已合并，最终实现了 HP ZBook Ultra G1a 笔记本电脑及其他未来高端 AMD Ryzen 笔记本电脑上的网络摄像头功能。

Intel EDAC 驱动为 Diamond Rapids 和 Nova Lake H 做准备。

- ARM64 方面没有太多进展，因为开发者被 AI 驱动的修复所淹没。

RISC-V 减少内核启动开销，并在 Linux RISC-V defconfig 中默认启用 ESWIN SoC 支持。

ARM64 NEON 的 CRC64 内联函数现在可在 32 位 ARM 上工作。

继续淘汰 i486 CPU 支持，该支持最初在 Linux 7.1 中被弃用/禁用。

引入 Panther Lake R，作为新的“加固型”Panther Lake SoC 版本。

各种电源管理改进。

Linux 内核优化：

一个不错的 MGLRU 改进，在使用补丁进行的基准测试中，MongoDB 的吞吐量提高了 30~100%。

更快的轮询性能和其他加速，在我们的测试中也观察到了一些本地网络加速。

缓存感知调度也可以为不同的处理器和不同的工作负载带来一些不错的改进。

更多分配器优化。

改进了匿名/未命名管道的性能，适用于 shell 管道等。

显著降低容器退出/卸载延迟。

更快的 /proc/filesystems 读取。

AMDGPU HDMI 2.1 FRL 支持，终于实现，朝着上游 AMDGPU 驱动中完整 HDMI 2.1 支持的最终目标迈进。

新的 AMDGPU DC 电源模块。

AMD 继续启用下一代图形 IP。

下一代 NPU 硬件。

Intel 为多个 Crescent Island SKU 做准备。

Intel DRM 背景颜色属性支持。

Nouveau 对 NVIDIA GA100 加速器的支持。

NVIDIA 开始为“Blackwell-Next”做准备。

其他各种 DRM 内核图形驱动改进。

Bcachefs 1.38.6 基准测试：首个不再标记为“实验性”的版本。

Intel Core Ultra X7 Panther Lake 在 Linux 7.1 上的性能。

缓存感知调度在 AMD Zen 5 上对 PostgreSQL、Valkey、网络性能展现出不错的效果。

Linux 6.6 LTS 到 Linux 7.1 基准测试：三年间 AMD Threadripper 性能提升 13%。

GCC 16 生成的二进制文件比 GCC 15 更快，与 LLVM Clang 22 竞争激烈。

Linux 7.1 特性：新 NTFS 驱动、新 Intel + AMD 硬件、性能优化与现代化。
