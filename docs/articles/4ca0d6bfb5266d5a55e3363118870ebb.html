NVIDIA 将初始 Rigel CPU 核心支持上游提交至 GCC 编译器

发布于 2026 年 7 月 7 日 美国东部时间上午 11:45。

这并没有花太长时间……就在 NVIDIA 确认了一些基本的 Rosa CPU 细节及其“Rigel”CPU 核心几分钟后，上游 GNU 编译器集合（GCC）代码库中已合并了对 NVIDIA Rigel 核心的初始支持。

NVIDIA 搭载 Rigel 核心的 Rosa CPU 是搭载 Olympus 核心的 Vera CPU 的继任者。NVIDIA 今天的博客文章确认了 Rigel 内部核心基于 Armv9.2-A 以及其他基本细节。随着这一确认的公布，NVIDIA 编译器工程师 Dhruv Chawla 成功将初始 CPU 目标上游提交至 GCC Git 代码库，用于定位下一代搭载 Rigel 核心的 Rosa CPU。但目前尚未包含任何专门的调优或其他可能稍后到来的更改。Olympus 的相同 ISA 能力及其他属性与 Rigel 共享。在识别 CPU 核心方面，Rigel 现在使用部件 ID 0x11，而 Olympus 为 0x10。GCC 中对 NVIDIA Rigel 的初始支持已通过此提交落地。

无论如何，很高兴看到在明年 GCC 17 稳定版发布之前，初始的底层支持已经合并到 GCC Git 中。希望在 GCC 17.1 之前，能对这个 Rigel CPU 目标进行进一步的调优。
