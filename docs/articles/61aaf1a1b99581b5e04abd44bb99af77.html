Linux 7.2 在默认内核构建中启用 UltraRISC RISC-V 支持

发布于 2026 年 7 月 12 日 下午 4:25 EDT

在 RISC-V "defconfig" 内核构建中默认启用 Eswin SoC 支持的同时，UltraRISC RISC-V 覆盖范围也已在 Linux 7.2 的 RISC-V 内核构建中默认启用。

在即将发布的 Linux 7.2-rc3 每周测试版本之前，今天合并了 RISC-V 修复，其中包括添加 Kconfig 支持以启用 UltraRISC RISC-V 硬件支持，在主线上游内核中目前对应 UR-DP1000（集成八个 UltraRISC C100 内核）。

该 UltraRISC RISC-V SoC 支持随后在 RISC-V defconfig 中默认启用。ARCH_ULTRARISC 是一个在合并窗口后较晚添加的选项，但已被部分驱动引用。

该支持不久前通过以下内容合并：

Linux 7.2-rc2 将默认 RISC-V 64 位 CPU 限制提升至 256 核

Linux 7.2 RISC-V 减少内核启动开销，默认启用 Eswin SoC 支持

Linux 7.2 将在 RISC-V 内核构建中默认启用 ESWIN SoC 支持

BeagleV Ahead 和 Lichee Pi 4a RISC-V 开发板将在 Linux 7.2 中获得可用的 WiFi

更多 SpacemiT K3 和 K1 支持进入上游 Linux 7.2

RISC-V BeagleV Ahead 单板计算机将在 Linux 7.1 中获得可用的 HDMI
