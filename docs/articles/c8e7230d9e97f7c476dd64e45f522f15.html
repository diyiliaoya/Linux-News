ELM11-Feather：原生支持Lua语言的Feather兼容板

ELM11-Feather是一款基于FPGA的Feather兼容开发板，原生运行Lua语言，无需传统微控制器。它允许用户在Lua、C和VHDL/SystemVerilog三个层面编程，支持23个可配置I/O引脚，售价39美元。该板通过GOWIN FPGA实现双核Lua REPL，并支持完整的硬件栈修改，兼容现有FeatherWing扩展板。BrisbaneSilicon已开启众筹，硬件原理图和固件API将在生产后以MIT许可证发布。

ELM11-Feather是一款原生支持Lua语言的Feather兼容板

它是一款基于FPGA的开发板，允许用户在Lua、C和VHDL/SystemVerilog中编程应用层、驱动层和硬件层，起售价29美元。

目前市场上大多数微控制器板依赖Python，通常以MicroPython的形式呈现。如果你想要更轻量级的方案，又不愿放弃无需编译、REPL驱动的工作流程，Lua是值得考虑的选择。但大多数微控制器板并非为Lua设计，即使你可以在上面运行Lua。

这正是总部位于布里斯班的小型半导体公司BrisbaneSilicon试图填补的空白——他们推出了一款原生运行Lua的ELM11-Feather板。实际上，这是该公司原版ELM11的Feather外形尺寸升级版，其众筹活动现已在Crowd Supply上线。

Lua（应用层）、C（驱动层）、SystemVerilog/VHDL（硬件层），全部集成在一块板上

GOWIN FPGA（无独立CPU核心，FPGA运行一切）

23个引脚，每个均可配置为GPIO、PWM、UART、SPI或I²C

尺寸：22.86 x 64.65 x 4.85 mm（0.9 x 2.54 x 0.191英寸）

兼容Feather，可与现有FeatherWing扩展板配合使用

ELM11-Feather售价39美元。

除了规格参数，架构选择才是亮点。这块板上没有传统的微控制器，而是由GOWIN FPGA完成所有工作，运行双核设置，每个核心上有一个独立的Lua REPL。这就是“巧妙之处”。由于Lua运行在FPGA而非固定MCU上，BrisbaneSilicon可以将整个栈（包括硬件）暴露给用户进行修改。

23个I/O引脚中的每一个都可以配置为GPIO、PWM、UART、SPI或I²C，与那些固定数量每种接口的板子相比，每个引脚提供了极大的灵活性。该板还配备了1 MB RAM、硬件看门狗、5个用户可编程LED以及一个内置的500 mA锂聚合物充电器（带状态LED），所有这些都集成在Feather紧凑的尺寸（22.86 x 64.65 x 4.85 mm）内，重量仅5.2克。兼容Feather意味着它可以无缝接入现有的FeatherWing扩展板生态系统，无需任何适配。

全栈可编程性

BrisbaneSilicon称之为“全栈可编程性”，这也是该板的主要卖点。其理念是同一产品可以在三个独立层面同时扩展：应用层运行Lua，驱动层运行C，硬件层通过可更换的“硬件覆盖层”运行VHDL/SystemVerilog。

在实践中，这意味着用户可以设计一个自定义硬件模块（例如正交编码器），为其编写C驱动程序，然后通过类似quadrature_encoder_speed()的普通函数将其暴露给Lua脚本。该工作流程无需为每个层面使用单独的工具链，因为BrisbaneSilicon自己的IDE（名为Arvore）旨在统一这三个层面。

Arvore通过一个界面处理项目创建、上传和扩展Lua API，测试版现已可供下载。对于不想手动安装自定义硬件覆盖层的用户，IDE也提供了配置界面。那些更愿意在命令行完成所有操作的纯粹主义者也不会被排除在外，命令行路径仍然可用。

BrisbaneSilicon表示，一旦众筹活动结束并开始生产，硬件原理图和固件API都将以MIT许可证发布。

与同领域其他几款开发板相比：

Adafruit Feather RP2350（RP2350 + ICE40UP5K）：GPIO和RAM更多，但无原生脚本语言，价格更高

Adafruit Feather STM32F405：更便宜，但无FPGA且无硬件层可扩展性

Adafruit HUZZAH32（ESP32）：电池续航更好，但I/O选项少得多，且无脚本到硬件的流水线
