Ubuntu 26.10 Snapshot 2 发布（含一项“破坏性变更”）

Ubuntu 26.10 的第二个快照版本 Snapshot 2 已发布，这是稳定版发布前四个快照中的第二个。本次更新没有太多新功能，主要面向开发者和 Bug 猎人。Canonical 警告了一项“破坏性变更”：Ubuntu cdimage 服务器布局已统一，桌面版镜像的 URL 路径发生变化，从顶级目录移至命名子文件夹，每日构建的 URL 也相应调整。快照是自动化构建的“一次性制品”，适合测试，但普通用户建议使用每日构建。

Ubuntu 26.10 Snapshot 2 已发布，这是“Stonking Stingray”开发周期中计划在 10 月稳定版发布前推出的四个快照中的第二个。

与第一个快照一样，这里没有太多“新”东西可看或测试，因此除非你是开发者或狂热的 Bug 猎人，否则没什么理由急着去尝试。

Canonical 的 Utkarsh Gupta 在 Ubuntu 开发者邮件列表中宣布该版本时，警告了一项“破坏性变更”——别慌：它不在镜像本身，而是访问镜像的 URL。

过去几周，Ubuntu cdimage 服务器布局已统一。Ubuntu 桌面版发行版已被移至一个命名子文件夹，以匹配 Ubuntu 官方衍生版所采用的方式。重定向处理了现有链接（因此本质上没有破坏任何东西）。

以前，Ubuntu 桌面版镜像位于顶级目录：

https://cdimage.ubuntu.com/releases/24.04/

现在它们位于命名文件夹中：

https://cdimage.ubuntu.com/ubuntu/releases/24.04/

衍生版没有实质变化，因为它们一直位于命名的顶级文件夹中：

https://cdimage.ubuntu.com/xubuntu/releases/24.04/

但真正“破坏”的是每日构建所使用的 URL，因为 Canonical 选择对指向它们的链接实施重定向。

这些链接不再位于：

cdimage.ubuntu.com/daily-live

而是位于：

cdimage.ubuntu.com/ubuntu/stonking/daily-live

衍生版的 URL 也发生了变化，增加了一个系列中间层，对于本周期来说是“stonking”。

Ubuntu 月度快照面向谁？

Ubuntu 的月度快照并非传统的 alpha 版本（该发行版已于 2013 年停止使用）。Ubuntu 的工程师将其描述为“一次性制品”（原文如此），因为它们是 Canonical 新的自动化镜像构建流水线生成的自动冻结版本。

快照没有经过同等程度的人工质量保证，但其准时性为贡献者提供了一个软目标来测试变更，也让 Canonical 有机会对新系统进行压力测试。

如果你是最终用户（而非开发者），并且只是想测试，最好获取每日构建——它们更新鲜。

Ubuntu 26.10 Snapshot 3 预计于 7 月 30 日发布。如果你确实安装了 Snapshot 2，只需在更新到来时安装它们，即可跟随开发浪潮。

面向用户的变化将在未来几个月内到来，包括 GNOME 51 桌面 alpha 版，以及 Ubuntu 计划中的 AI 功能中的第一个——在任何文本字段中进行语音输入。

从 Ubuntu cdimage 服务器（使用新 URL）下载 Ubuntu 26.10 Snapshot 2。
