Ubuntu 将 Livepatch 引入 arm64，实现免重启内核更新

Canonical 首次将 Livepatch 功能引入 Arm64 设备，使运行在 Arm 硬件上的 Ubuntu 系统能够在不重启的情况下应用关键内核安全补丁。Livepatch 是 Ubuntu 隐藏的安全特性之一，需通过 Ubuntu Pro 启用。该功能此前在 arm64 架构上因堆栈跟踪支持不足和工具链缺失而难以实现，经过 Canonical、内核维护者、硬件厂商和超大规模云服务商多年的协作才得以完成。目前支持的 arm64 内核变体包括 Arm 服务器、云和始终在线硬件。但需注意，Livepatch 仅更新内核，常规软件包仍需通过标准流程更新，且 Canonical 建议定期重启。

Ubuntu 将 Livepatch 引入 arm64，实现免重启内核更新 - OMG! Ubuntu

Canonical 首次将 Livepatch 功能引入 Arm64 设备，使运行在 Arm 硬件上的 Ubuntu 系统能够在不完全重启的情况下应用关键内核安全补丁。

Livepatch 是 Ubuntu 最隐蔽的安全特性之一——它默认未启用，需要 Ubuntu Pro——因为它允许在系统运行时在内存中应用内核安全更新。通常情况下，这需要重启。

如果你有点懒，或者正在运行不想被打断的任务或工作负载，这非常完美。

Livepatch 现已首次在 Ubuntu 26.04 LTS 及更高版本上可用于 Arm64 设备——不过让一切正常工作并非易事。

“实时”打补丁需要内核知道何时可以安全地将运行中的代码替换为修补后的版本。这依赖于堆栈跟踪，而 arm64 对此支持不佳。用于在 arm64 上构建和比较补丁的工具链也同样缺乏。

这需要 Canonical、内核维护者、硬件厂商和超大规模云服务商之间多年的协作努力才得以实现。

我可以在树莓派上使用 Ubuntu 的 Livepatch 吗？

如果你在树莓派上运行 Ubuntu 26.04 LTS，你将无法使用 Livepatch。

Ubuntu 的树莓派版本运行在树莓派内核上，这是一种内核变体，不在 Livepatch 支持的列表之中。

Arm 驱动的服务器、云和始终在线硬件是 Livepatch 的用武之地，确保关键和高严重性的 CVE（那些可能需要非计划重启才能应用的漏洞）能够在不宕机的情况下原地修补。

“这一增强将加强那些不每天或每周进行安全维护的系统的安全性，并为致力于符合《网络弹性法案》（CRA）的组织提供运营优势，”Canonical 的 Rajan Patel 写道。

不过，这并非完全无需动手的安全方案。Livepatch 仅更新内核。常规软件包（如 OpenSSL 等）仍需通过标准更新流程进行维护。Canonical 也建议定期重启。

在 Ubuntu 26.04 LTS 上启用 Livepatch

要使用 Livepatch，你需要一个 Ubuntu Pro 账户，个人使用最多五台机器免费。Ubuntu 26.04 LTS 用户可以通过桌面端的软件更新器或在无头设置、服务器和云上通过命令行启用它。
