Linux 7.2-rc2 发布：“情况非常正常”

2026年7月5日晚上9点08分（美国东部时间）

Linux 7.2-rc2 现已可供测试，以推进稳定版 Linux 7.2 的开发。本周的 Linux 7.2-rc2 将默认的 RISC-V 64 位核心限制从 64 提升至 256，以更好地反映 RISC-V 设计日益增长的核心数量。Linux 7.2-rc2 还加强了对 BPF 的防护，以抵御 JIT 喷射攻击。此外，还有大量其他更新，主要是常规的错误和回归修复。

Linus Torvalds 写道：

“现在是周日下午，rc2 发布了。情况看起来非常正常——这不是一个小型的 rc2，但与前几个版本一致，且略小于 7.1 的 rc2。让我们看看后续如何发展，但到目前为止一切顺利。

diffstat 看起来也很正常，大约一半的补丁是驱动程序——通常的嫌疑对象占据了大部分。其余补丁分布在各个领域（但主要是文件系统和网络）。

唯一稍微不寻常的是头文件拆分，mod_devicetable.h 被拆分为多个更小的、按子系统划分的设备 ID 头文件，这本应早就如此处理，但之前没有做到。这是一次清理，同时也意味着我们少了一个巨型头文件——这类头文件一旦被修改，几乎会导致所有内容重新编译，即使改动只涉及少数几个小文件。

无论如何，这在 diffstat 中有所体现，但希望在其他方面不会明显可见，因为所有用户都应该已经正确更新了。敲木头。”
