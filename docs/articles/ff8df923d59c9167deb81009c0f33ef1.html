Linux 7.3 将克服 PCIe Gen5 NVMe SSD 小 I/O 的“重大瓶颈”

发布于 2026 年 7 月 1 日 12:06 PM EDT。

Linux 7.2 的特性合并窗口几天前刚刚结束，距离 v7.2 稳定版发布还有近两个月的时间，但针对 Linux 7.3 周期的特性已经开始积累。我看到的开启 Linux 7.3 前奏中最令人兴奋的改动，是解决了一个影响快速存储（如 PCIe Gen5 NVMe SSD）小规模直接 I/O 性能的“重大”瓶颈。

字节跳动工程师 Fengnan Chang 注意到，在 Gen5 NVMe SSD 上执行 4K 随机读取时存在一个“重大瓶颈”。显著的软件开销被追溯到 Linux 内核的 IOmap 框架及其直接 I/O 路径。CPU 最终花费大量时间在内存分配和 IOmap 状态机上。为此，Fengnan Chang 设计了一个简单的直接 I/O 路径，用于在使用 IOmap（例如 EXT4 和 XFS）时处理小规模直接 I/O。

这个简单的 DIO 路径用于减少 IOmap 开销，当读取请求的 I/O 大小小于或等于 inode 块大小时生效。此外还有其他限制条件，例如 inode 未加密。经过这一创建，在 PCIe Gen5 NVMe SSD 上的测试中，4K 随机读取性能从 1.92M IOPS 跃升至 2.19M IOPS。额外的基准测试显示，在使用 IO_uring 和现代 PCIe Gen5 NVMe SSD 存储时，尤其是在较高队列深度下，EXT4 和 XFS 的性能提升高达 10%。

这个用于 IOmap 的简单 DIO 路径已合并到 VFS 子系统的“fvfs-7.3.iomap”Git 分支中。由于它现在已被排入 vfs/vfs.git 的“7.3”分支，预计将在今年晚些时候的 Linux 7.3 周期中提交。
