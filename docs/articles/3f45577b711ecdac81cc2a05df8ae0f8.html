为何你的Ubuntu 26.04对更新保持沉默

Ubuntu 26.04 LTS默认关闭了更新通知托盘图标，导致用户不再收到桌面更新提醒。尽管系统仍会在后台自动安装安全更新，但关闭该功能后用户将完全无法通过桌面获知更新状态。这一设计选择引发争议，用户可通过终端命令手动重新启用图标。文章提醒用户应养成自行检查更新的好习惯。

以下是为何你的Ubuntu 26.04对更新保持沉默的原因 - OMG! Ubuntu

已经有一段时间没有就待处理的更新烦扰你了，这背后是有原因的。

在Ubuntu Discourse上，有人注意到该发行版不再在更新待处理时发出通知——前提是更新通知托盘图标已启用。

问题在于？这个托盘图标在Ubuntu 26.04中被默认关闭了（且没有提供切换开关）。

不过，这并不像听起来那么“危险”。

Ubuntu 26.04默认启用了自动更新，它会在后台安装安全更新——只是你不会被告知而已。但如果你关闭了自动更新，情况就变得棘手了：bug报告指出，你将永远不会通过桌面收到任何更新的通知。

在我看来这算是一个bug，但它也很好地提醒了我们，无论默认设置如何，保持良好的系统习惯（自行检查更新）至关重要。

如果你想及时了解更新情况，可以在26.04中重新启用这个托盘图标。

过去，如果有可用更新，这个图标会定期弹出，但许多人（包括我自己）觉得它很烦人，因为那个窗口常常会抢走焦点。

Ubuntu 25.10修复了这个问题，通过添加桌面通知和托盘图标来安静地通知更新。

但那个托盘图标在Ubuntu 26.04 LTS中被默认关闭了。Canonical的Marco Trevisan表示，这“是一个设计选择”，人们可以对此争论，但该托盘图标可以通过以下方式手动重新启用。

再次强调：Ubuntu 26.04会自动安装更新，你也可以手动操作：运行`apt upgrade`，因为它已经获得了同时升级Deb软件和Snap的能力。

但那些习惯了被提醒和催促更新的老Ubuntu用户，可能会觉得突然的沉默有点令人不安（或者让人松一口气；无知是福嘛）。

自己启用那个缺失的图标很简单，只需打开终端运行：

`gsettings set com.ubuntu.update-notifier show-updates-status-icon true`

（如前所述，设置中没有提供切换开关）。

你也可以使用GUI工具`dconf-editor`编辑该值，但很容易不小心改错一个你本不想改的设置，尤其是在使用鼠标滚轮浏览页面时（亲身经历）。

Ubuntu 26.04从新安装中移除了`update-manager`，以使系统管理不那么晦涩。该工具仍然可以从软件源安装，因此如果你想对更新的安装时间、方式或是否安装有更多控制权，值得把它装上。
