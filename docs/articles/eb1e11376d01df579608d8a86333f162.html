Linux 7.2 在 AMD EPYC Sorano 上展现出一些意外且显著的性能提升

2026年6月23日 美国东部时间下午12:22

合并窗口要到本周末才结束，作为新功能特性的截止点，但我已经开始对当前为这个下一版 Linux 内核暂存的代码进行一些早期基准测试。Linux 7.2 已经因缓存感知调度和其他令人兴奋的新特性而显得相当引人注目，而本周早期测试中的一个意外惊喜是看到了一些本地网络/套接字性能的改进。

我用于评估 Linux 7.2 内核早期性能的第一个系统是一台搭载全新 AMD EPYC 8005 "Sorano" 系列处理器的服务器，具体来说是旗舰级的 84 核/168 线程 Zen 5 处理器。Linux 7.2 在 EPYC Sorano 上表现良好，我很快会提供内核性能的完整概览，以及其他硬件上的基准测试，但在这篇初步文章中，意外的惊喜是发现本地主机网络性能有了显著提升。

再次提醒，请继续关注合并窗口结束后更全面的 Linux 7.2 基准测试，而今天先简要探讨一下让我感到惊讶的网络性能提升。

在将 Linux 7.1 稳定版与截至 6 月 21 日的 Linux 7.2 Git 版本进行比较时，测试这台 EPYC Sorano 服务器的本地主机性能时，意外地发现了一些令人愉快的改进，测试工具是 xfr，一款基于 Rust 的现代 iperf3 替代品：

TCP 性能提升非常显著。UDP 性能与 Linux 7.1 相比没有变化。

在 Linux 7.2 开发版内核上运行的 xfr QUIC 测试相比 Linux 7.1 稳定版也显示出不错的改进。这对 AMD EPYC 8005 "Sorano" 面向电信、边缘计算、云存储等环境来说是个好消息。在接下来的测试中，看看这些改进是否同样适用于其他硬件，将会很有趣。

除了 xfr 之外，Sockperf 网络套接字基准测试也显示 Linux 7.2 的延迟更低。

对于 Linux 7.2 中这些意外改进的一个可能解释，在运行 stress-ng 内核微基准测试时浮出水面。Linux 7.2 Git 版本的 poll 性能比 Linux 7.1 稳定版快了数倍。

在内核中实现 poll 的地方，Linux 7.2 进行了调整，将 poll 代码从 free_page() 切换到了 kmalloc() API。这一改动没有注意到任何性能数字上的变化，但确实表明使用 kmalloc API 会提供更好的可扩展性，而这正是我们在这些基准测试中可能看到的。这是一个通用性的改进，并非 AMD 或 EPYC 特有，因此我们应该也能在其他硬件上看到收益。

同样重要的是，发现 AMD EPYC 8635P CPU 在 Linux 7.2 上取得这些基准测试胜利的同时，功耗并没有增加，反而比 Linux 7.1 略有降低。

无论如何，请继续关注合并窗口结束后更多的 Linux 7.2 内核基准测试，而目前这只是本周在尝试使用这个开发中的 Linux 内核测试全新 AMD EPYC Sorano 时遇到的第一个意外惊喜。
