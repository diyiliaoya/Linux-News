GraalVM CE 25.1.3 将 Native Image “Hello World” 程序缩小至仅 6.5MB

发布于 2026年6月30日 下午4:11 EDT。

GraalVM 是一款专注于提前（AOT）Native Image 编译的高级 JDK，自去年起开始将重心转向更多非 Java 语言（如 Python 和 JavaScript）。如今，它迎来了最新的社区功能版本。GraalVM Community Edition 25.1.3 现已发布，带来了一些有趣的变化。

GraalVM 25.1.3 增加了对使用 Graal 编译器录制和回放 JIT 编译的支持，新增了一个反汇编工具，并提供了针对不可信代码的可选常量盲化支持。此外，还有性能改进、调试增强、兼容性提升、Linux 构建默认生成位置无关可执行文件（PIE），以及其他增强功能。同时，作为 JavaScript 实现的 GraalJS 和作为 Python 实现的 GraalPy 也获得了更多语言支持，功能更加丰富。

GraalVM 25.1.3 还引入了 Web Image 作为 GraalVM Native Image 的一个新的实验性后端，可将 Java 应用提前编译为带有 JavaScript 包装器的 WebAssembly 模块。

最让我惊讶的一项变更，是通过进一步压缩来减小 Native Image 体积：

“通过压缩镜像堆元数据和存储（包括模块元数据、运行时动态访问元数据和符号表数据），并对简单的常量 String.format 和 String::formatted 调用进行内联优化，从而减小了 Native Image 体积。这使得 Linux AMD64 上的 HelloWorld 应用体积缩小至约 6.5 MB。”

也就是说，让我惊讶的是，一个基本的“Hello World”程序竟然能缩小到……

也许是我对其他语言太习惯了，但一个“Hello World”程序生成 6.5MB 的二进制文件？！我很好奇之前 HelloWorld AMD64 Linux 构建的体积有多大，于是翻查了相关活动记录。Graal 的大部分开发工作都在 Oracle 闭门进行，但有人就体积影响提出了公开问题。本次版本中额外的 Native Image 压缩“整体上应能使镜像体积减少约 1-2%。”

“总体而言，与 25.0.x 版本相比，25.1 版本在镜像体积上会有更大的改进，因为我们在这一领域进行了多项较小的优化（有些仍在进行中），这些优化累积起来效果显著。同时，由于更紧凑的表示方式，RSS 也有较小的正面影响。由于 String format 的内联优化，HelloWorld 镜像体积会出现非常大的下降。”

但即便如此，将 HelloWorld 应用缩小到 6.5MB；是我太挑剔，还是对于大多数“hello world”程序来说，这仍然相当臃肿？
