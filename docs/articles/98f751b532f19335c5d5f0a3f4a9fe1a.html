KDE Linux 引入“开发者模式”选项，简化日志收集

2026年7月1日下午5:06 EDT。

随着新月的到来，KDE Linux 发行版发布了上个月的进展报告。尽管 KDE 开发者在六月忙于发布 Plasma 6.7，他们在 KDE Linux 方面仍然取得了不少成就。

KDE Linux 推出了初始“开发者模式”，用于显示各种默认隐藏的开发者工具和选项，以简化最终用户体验。只需在 Konsole 中运行 toggle-developer-mode 即可启用这个初始开发者模式。

这种易于激活的开发者模式为非开发者用户提供了简洁干净的体验，他们偶尔可能需要切换额外功能来帮助调试问题。目前，toggle-developer-mode 功能相当基础，但未来可能会在此基础上进一步定制开发者体验。

如果你在2026年还在翻录 CD，KDE Linux 现已切换到 Audex 应用，替代 audiocd-kio 用于 CD 翻录。KDE Linux 开发者已成功将 Audex 打包到 Flathub 上。

六月，KDE Linux 在 QA/测试基础设施方面也取得了更多进展，将 KDE Linux 的 RAW 镜像文件转换为混合 ISO（同时也是有效的 ISO 文件），通过新工具简化日志收集，并改进了文档。collect-logs 工具灵感来自 CachyOS 的 bug 报告脚本，用于轻松收集 kscree-doctor、uptime、/etc/fstab 等命令和常见文件的输出。

关于 KDE Linux 在六月期间改进的更多详情，可参见 KDE 开发者 Nate Graham 的文章。
