libx11-compat 正致力于在 SDL 之上实现 Xlib，以支持 Wayland、macOS 和 Android

发布于 2026 年 7 月 22 日，美国东部时间上午 8:28。

为了帮助 X11 客户端在采用 Wayland 的现代 Linux 桌面环境中运行，甚至能在 macOS 或 Android 上运行，现在又多了一个正在开发中的选项——libx11-compat。libx11-compat 项目正致力于在 SDL2/SDL3 接口之上重新实现 Xlib 客户端库，以实现广泛的底层平台覆盖。

开源开发者 Ching-Chun (Jim) Huang 撰文介绍了他在 libx11-compat 上的工作，该项目旨在为 X11 客户端提供一个轻量级兼容层。对于依赖 Xlib 的应用程序，无需修改源代码即可运行，例如在无需 XQuartz 的 macOS 上、纯 Wayland 的 Linux 桌面会话中、无头 CI 环境、Android 以及其他非 X11 原生平台上。

libx11-compat 库是 X Window 系统客户端库的进程内实现，构建于 SDL2/SDL3、SDL_ttf 和 Pixman 之上。该代码目前无需 X 服务器即可运行大量经典 X11 应用程序，如 XMosaic、Qt2 应用、XCircuit、XMMS 等。
