AMD 在 Linux 上启用 CACP 功能以实现更佳的 OLED 省电效果

AMD 在最新的 AMDGPU DC 显示代码更新中，为 Linux 开源驱动启用了“CACP”（内容自适应对比度与功耗）功能，旨在进一步降低 OLED 显示面板的功耗。该功能属于 AMD 自适应背光管理（ABM）的一部分，已在 Windows 驱动中应用多年。此外，本次更新还启用了面板自刷新（PSR）和面板重放功能，并修复了 8K 模式解析问题。这些补丁预计将进入 Linux v7.3 内核周期。

AMD 在 Linux 上启用 CACP 功能以实现更佳的 OLED 省电效果

发布于 2026 年 7 月 9 日，美国东部时间晚上 8:51。

今天发布的 AMDGPU 显示代码“DC”更新带来了一些值得关注的改进，使现代硬件在 Linux 下受益。

新一批 AMDGPU DC 代码更新针对其开源 Linux 驱动，带来了一些值得注意的内容，同时还有大量测试用例的添加，以期帮助保持质量并防止未来的回归问题。

在今天 AMDGPU DC 补丁的变动中，引起我注意的是在 Linux 上启用了“CACP”。据称这有助于 OLED 显示面板在 Linux 上节省更多电量。此处的 CACP 是“内容自适应对比度与功耗”（Content Adaptive Contrast and Power）功能的缩写，属于 AMD 自适应背光管理（ABM）处理的一部分。

补丁消息或代码注释中并未提供量化相对省电效果的数据。搜索“AMD Content Adaptive Contrast and Power”也没有太多发现。无论如何，看看这是否能为 Linux 下的 OLED 显示面板使用带来显著的省电效果，将会很有趣。

代码中的一些版权注释反映了 AMD 2022 年的代码，因此这项省电功能似乎早已在其 Windows 驱动中使用。

今天的 DC 补丁还在 DCN42b 变体上启用了面板自刷新（PSR）和面板重放功能，以针对该特定显示 IP 实现更大的省电效果。IPS 支持在另一个针对 DCN42b 的补丁中作为另一项省电功能被启用……这似乎是今天补丁发布在 Linux 上的一个重点。

在这些补丁中，还有一个针对 8K 模式解析的修复，适用于显示 EDID 未解析的情况。由于 AMDGPU 的处理方式，某些 8K@120 和 8K@240 时序可能无法通过作为 HDMI 2.1 处理一部分的 DisplayID 扩展块正确解析。新补丁正在修复此问题。

这些新补丁已经发布。考虑到时间节点，这些补丁很可能会被纳入 Linux v7.3 内核周期。
