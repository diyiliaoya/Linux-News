Fedora 45 考虑默认启用 x86_64 Shadow Stack

Fedora Linux 45 正考虑一项变更提案，默认启用 x86_64 架构的 Shadow Stack（影子栈）保护，以提升现代 Intel 和 AMD 系统的安全性。该功能通过硬件强制保护返回地址，防御面向返回编程（ROP）攻击。默认情况下，动态链接器或静态启动例程将为所有二进制和共享库依赖均支持 Shadow Stack 的进程激活该保护。此变更向后兼容，性能开销极小，并为未来启用完整控制流强制技术（CET）奠定基础。

EFS文件系统在无人维护20多年后，计划于Linux 7.3中移除

EFS文件系统曾用于SGI IRIX系统的非ISO9660 CD-ROM和磁盘分区，但在IRIX 6.0转向XFS后被取代。Linux内核中的EFS只读驱动已超过20年无人维护，尽管有人曾表示愿意接手，但因其仅贡献了基础编译/错误修复且从未实际使用过EFS，内核开发者最终决定移除该驱动。该移除已合入vfs-7.3.efs分支，预计随Linux 7.3发布。

Linux内核开发者再次讨论AI代理归属标注——可能将其移除

Linux内核开发者近期再次讨论AI/LLM代理在补丁中的归属标注政策。当前要求使用“Assisted-by”标签并注明模型版本等详细信息，但知名开发者Christian Brauner认为这相当于为AI公司提供免费广告，建议简化或完全取消该要求。内核工程师Jeff Layton已提交补丁提议彻底移除该标签，理由包括遵循率低、信号噪声比差、目的不明确等。Linus Torvalds尚未表态，讨论仍在进行中。
