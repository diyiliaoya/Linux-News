Ubuntu 26.10 在运行22年后将 dbus-daemon 退役为默认

Ubuntu 26.10 将自2004年以来首次更换其 D-Bus 实现，用 dbus-broker 取代 dbus-daemon。这一变化对终端用户几乎无影响，因为 dbus-broker 是即插即用的替代品，使用相同的协议、配置文件和总线策略。新实现采用异步事件驱动设计，能避免消息丢失并加快查找速度。此前因 Ubuntu 政策不允许重复包共存以及 AppArmor 兼容性问题，切换延迟至今。用户可在 Ubuntu 26.04 LTS 上手动安装测试。

LLVM 合并 x86 LFI“轻量级故障隔离”目标，用于进程内沙箱

斯坦福大学研究人员为 LLVM 开发了轻量级故障隔离（LFI）编译器和目标，用于高效的原生代码沙箱。此前 AArch64 LFI 目标已上游合并，本周 x86/x86_64 LFI 目标也完成合并。LFI 是一种基于编译器的进程内沙箱方法，旨在以最小性能开销和工程成本，为 C、C++ 和汇编代码提供内存安全保护。相比基于进程的沙箱，LFI 上下文切换仅需数十个周期，速度提升约 100 倍；相比 Wasm 工具链，开销降低 3-10 倍。在 Spec2017 上，完整沙箱仅带来约 7% 的开销（AArch64），仅保护写入时开销为 2%。
