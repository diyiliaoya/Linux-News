Linux 7.3 将克服 PCIe Gen5 NVMe SSD 小 I/O 的“重大瓶颈”

Linux 7.3 将引入一项重要优化，解决 PCIe Gen5 NVMe SSD 在执行 4K 随机读取时遇到的“重大瓶颈”。字节跳动工程师 Fengnan Chang 发现，Linux 内核的 IOmap 框架在直接 I/O 路径中存在显著的软件开销，导致 CPU 花费大量时间在内存分配和状态机上。通过为小规模直接 I/O 设计一条简化路径，4K 随机读取性能从 1.92M IOPS 提升至 2.19M IOPS，EXT4 和 XFS 文件系统在 IO_uring 场景下性能提升高达 10%。该优化已合并至 VFS 子系统的“fvfs-7.3.iomap”分支，预计将在今年晚些时候的 Linux 7.3 周期中提交。
